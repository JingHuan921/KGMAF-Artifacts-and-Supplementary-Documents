# KGMAF-Artifacts-and-Supplementary-Documents

Artifacts and Supplementary Documents of the KGMAF platform.

This repository contains all the PDF files of artifacts and supplementary documents generated by the KGMAF (Knowledge-Guided Multi-Agent Framework) platform.

## Agent-to-Document Mapping

The table below shows which agent produces which document in the KGMAF pipeline.

### Core Artifacts

| Agent | Document | Description |
|-------|----------|-------------|
| Interviewer | Product Analysis Report (PAR) | Summary and plan derived from user interview |
| Interviewer | User Requirement List (URL) | Structured list of user requirements |
| Interviewer | Interview Record (IR) | Persisted record of the interview conversation |
| Deployer | Operating Environment List (OEL) | Deployment environment, constraints, and dependencies |
| Analyst | Requirements Classification | Classified user requirements by category |
| Analyst | System Requirements | Formal system requirements list (SRL) |
| Analyst | Requirements Model | Use case diagram (UML) |
| Archivist | Software Requirement Specs (SRS) | Full SRS document following IEEE 830 / ISO 29148 |
| Reviewer | Review Document | RAG-augmented review comments on the SRS |
| Reviewer | Validation Report | Consolidated validation report with pass/fail status |

### Supplementary Documents

| Agent | Document | Description |
|-------|----------|-------------|
| Business Analyst | User Class | Identification of user roles, goals, and interaction patterns |
| Business Analyst | Feature Tree | Hierarchical organization of system features |
| Business Analyst | Business Scope | Project scope, objectives, and constraints |
| Business Analyst | User Stories | User stories and use cases for each user role |
| Business Analyst | Business Requirement Document (BRD) | Consolidated business requirements |
| Analyst | Dialogue Diagram | Sequence diagram of system interactions |
| Analyst | Data Flow Diagram (DFD) | Data flow between system components |
| Analyst | ER Diagram | Entity-relationship diagram for data model |
| Analyst | Data Dictionary | Definitions of data entities and attributes |

## Workflow Overview

```
User Input
  |
  v
Business Analyst (supplementary chain)
  User Class -> Feature Tree -> Business Scope -> User Stories -> BRD
  |
  v
Interviewer
  PAR -> Interview -> URL -> IR
  |
  v
Deployer
  OEL
  |
  v
Analyst
  Requirements Classification
    |
    v
  Analyst (supplementary chain)
    Dialogue Diagram -> DFD -> ER Diagram -> Data Dictionary
    |
    v
  System Requirements -> Requirements Model
  |
  v
Archivist
  SRS Generation -> SRS Revision (if needed)
  |
  v
Reviewer
  Review Document (RAG-augmented) -> Validation Report
  |
  v
  [Pass] -> Done / User chooses next action
  [Fail/Issues] -> Auto-revise SRS -> Re-review (up to 5 cycles)
```
